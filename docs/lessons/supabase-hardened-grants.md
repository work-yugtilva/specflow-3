Modern supabase postgres images grant API roles (anon/authenticated/service_role) NO DML on new public tables — default ACLs give only TRUNCATE/REFERENCES/TRIGGER/MAINTAIN. Every migration creating tables must self-grant: `grant select, insert, update, delete on all tables in schema public to authenticated, service_role;` + sequence usage. Order grants BEFORE append-only revokes or the revoke is undone. service_role keeps BYPASSRLS; `postgres` role is NOT superuser — `set role mcp_reader` needs explicit `grant mcp_reader to postgres`.
